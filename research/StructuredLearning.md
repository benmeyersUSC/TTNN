# Structured Learning

## Core Question

When a dataset is generated by an underlying rule (i.e., when the data has structure), does a
network forced to learn intermediate representations of that rule learn faster, generalize better,
and require less capacity than one that only sees final outputs?

The hypothesis is yes — and the claim is not just about final quality, but about sample efficiency,
convergence speed, and the internal economy of the learned model.

---

## Motivation: Kolmogorov Complexity

Structured data has a short generating program — its Kolmogorov complexity is low. A learner that
discovers the generating program compresses the data. A memorizer treats the data as if it were
random: its minimum description length equals the data length, no compression achieved.

The intermediate tap heads are forcing the network to build up the generating program incrementally.
Each tap corresponds to an intermediate step in the rule's computation. A network trained this way
is pushed toward the short description of the data. An untapped network that only sees final outputs
must either stumble onto the generating program or fall back on memorization.

This is the formal backbone of the intuition: learning the rules of something is literally finding a
compressed representation of the data. Memorizing is leaving information uncompressed — treating it
as random.

---

## Experimental Design

### Two Networks

**Network A — Memorization Baseline**
- Standard architecture, trained only on `(input → final output)` loss
- No intermediate supervision; must discover or memorize

**Network B — Tapped / Structured**
- Same architecture, but with intermediate heads at each rule-step depth
- Loss = final output loss + Σ tap losses (weighted)
- Implemented via `BranchTrainer` in TTTN — each tap is a branch head whose gradient flows back
  into shared trunk weights

### Dataset

Generated from a known rule with explicit intermediate steps (e.g., symbolic arithmetic,
context-free grammar derivation, multi-step logical inference). Ground truth at every
intermediate step is available at generation time — supervision is free.

### Metrics

1. **Sample efficiency** — examples required to reach threshold accuracy
2. **Generalization** — held-out accuracy on out-of-distribution inputs
3. **Capacity** — smallest model that reaches threshold (sweep over sizes)
4. **Gradient budget per source** (see below)
5. **Depth-weighted gradient work per source** (see below)

---

## Measurement: Gradient Budget Per Tap

An extension of the gradient budget metric from `TransformerTrainer`. Instead of two sources
(CE, RL), there are K+1 sources: the final output head plus each intermediate tap head k.

```
budget_k = N_updates_k × lr_k
```

Visualized as a stacked area chart over training time — shows what fraction of total gradient
influence was "purchased" by the final head vs. each intermediate tap. The tapped network
should show a meaningful fraction of budget going to intermediate heads early in training,
then shifting toward the final head as the intermediate representations stabilize.

---

## Measurement: Depth-Weighted Gradient Work

The gradient budget measures influence *available* to each source. This metric measures influence
*actually administered* — the cumulative L2 work done on weights by each source, weighted by
where in the network that work lands.

### Layer Leverage

A weight in layer l=1 (closest to input) participates in computing every subsequent layer's
input. A 1% change there reshapes all downstream computation. A weight in the final layer only
affects the logits. Define leverage:

```
leverage_l = (L - l + 1) / L
```

where l=1 is the input-adjacent layer and l=L is the output layer.

Note the subtlety: in the backward pass, early layers often receive *smaller* raw gradients
due to vanishing gradient attenuation (chain rule multiplies small Jacobians). So early layers
may have high forward leverage but low raw gradient energy. The depth-weighted metric captures
both: gradient energy × leverage = effective influence deposited.

### Depth-Weighted Gradient Work

For each loss source s, accumulated over training:

```
GradWork_s = Σ_t Σ_l leverage_l × ||∂L_s/∂W_l(t)||_F × lr
```

This gives a scalar per source per epoch representing where in the network each gradient source
is doing its work and how leveraged that work is. A source that concentrates gradient energy in
early layers does more leveraged work than one that only adjusts the final projection.

**Implementation**: in a multi-loss setup, each `.backward()` call is cleanly attributable to
one source. Per-layer gradient norms can be sampled periodically (not every step) and
accumulated. This can be logged alongside the gradient budget without significant overhead.

## Visualization Goals

Across training, for both networks side by side:

1. **Loss curves** — final head loss, each tap loss, total loss
2. **Gradient budget chart** — stacked area: fraction of budget per source over time
3. **GradWork chart** — cumulative depth-weighted gradient work per source over time
4. **Generalization gap** — train vs. test accuracy, showing memorization pressure in Network A
5. **Compression proxy** — how well activations at each layer can be predicted from the ground-truth
   intermediate rule output (measures how well the tapped net's layers align with the rule structure)

## Ablation: The Gradient Work Lever

Rather than one tapped vs. one untapped network, sweep the *amount* of Gradient Work done by the
tap heads by controlling their learning rate independently of the final head.

Define:

```
α = GradWork_taps / GradWork_total  ∈ [0, 1]
```

This is the lever. Run an array of training experiments across α:

### The Endpoints

**α = 0 — Pure Memorization**
Final head lr is nonzero, all tap lrs are 0. No intermediate supervision reaches the weights.
This is Network A — the baseline. The model maps x → y with no structural guidance.

**α = 1 — Full Tap Control**
Tap lrs are nonzero, final head lr = 0. The model is trained exclusively on intermediate
representations and never directly on the final task.

This is the most interesting ablation: it tests the Kolmogorov argument in its purest form.
If the model has learned the generating program (all intermediate steps), can it compose those
steps to solve the final task zero-shot — without ever seeing final-output supervision? If yes,
that is a very clean empirical result: learning the program was sufficient; the answer came for
free.

### The Interior

α ∈ (0, 1) — sweep in increments (e.g., 0.1 steps). The hypotheses are:

1. **Goldilocks zone**: there exists an optimal α (or range) where final-task accuracy and
   sample efficiency are both maximized — better than either extreme.
2. **Convergence speed**: even small α (a little tap Work) measurably accelerates convergence
   on the final head vs. α = 0. Structural scaffolding helps even when the final task dominates.
3. **Goldilocks curve**: plot final accuracy, generalization gap, and examples-to-threshold
   against α. Expect a curve with a clear peak interior to [0, 1], not monotone in either
   direction.

### Why the Lever Works

Controlling lr per loss source gives clean, interpretable control over the Gradient Work each
source deposits. The GradWork metric then serves as the x-axis of the ablation — not "what lr
did we set" but "how much leveraged work did each source actually do." This decouples the
experimental variable (Work fraction) from the implementation knob (lr ratio), and makes the
correlation claim direct: final task quality correlates with tap GradWork, not just tap lr.

## Connection to Human Learning

The structured/tapped network is the student who attends class, engages with derivations, and
builds an object of knowledge incrementally. The untapped network is the student who skips to
the answers and relies on memorization.

The empirical prediction: the tapped network reaches the same final accuracy faster, with fewer
parameters, and generalizes better — because it found the short program, not the lookup table.

The gradient budget and GradWork metrics make the mechanism visible: you can literally watch
how the intermediate supervision redirects gradient work toward foundational representations,
and trace that to the generalization advantage.
